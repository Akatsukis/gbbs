Definitions

Here we specify the definitions of objects and terms used in our benchmark specifications.

#Unweighted Graphs

We consider both undirected (symmetric) and directed (asymmetric) graphs in GBBS. We refer to graphs as G=(V,E)G=(V,E)G=(V,E), where VVV is a set of vertices, and EEE is a set of edges. Each edge consists of a pair of vertices (u,v)(u,v)(u,v).

When not specified, we assume that the underlying graph is directed. The concrete formats supported by GBBS for unweighted graphs can be found in the formats page

#Weighted Graphs

We also consider weighted graphs in GBBS, which we refer to as G=(V,E,w)G=(V,E,w)G=(V,E,w) where www is a weighted function w:E↦Rw : E \mapsto \mathcal{R}w:E↦R. We refer to the weight of a (u,v)(u,v)(u,v) edge as wuvw_{uv}wuv​ The benchmarks always specify the range, R\mathcal{R}R of the weight function, unless the algorithm is oblivious to it.

#Distances

We use distG(u,v)\mathsf{dist}_{G}(u,v)distG​(u,v) to refer to the shortest path distance between uuu and vvv in the graph GGG.

We use diam(G)\mathsf{diam}(G)diam(G) to refer to the diameter of the graph, or the longest shortest path distance in GGG between any vertex sss and any vertex vvv reachable from sss.

#Mapping

Many of our benchmarks return α\alphaα-mapping, which are objects that provide access to α\alphaα's over a specified input domain. For example, an int-mapping for the domain [0,n)[0, n)[0,n) can be represented

  • using an array of nnn ints, or
  • using a function from the domain to an int (implicitly represented).

The only requirement is that the returned object implements an array operator, and can be accessed at every index in the domain.
